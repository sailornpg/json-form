# Domain Docs

说明工程类 skill 在探索本仓库时应如何读取领域文档。

## 探索前优先读取

- 仓库根目录的 **`CONTEXT.md`**，如果它存在。
- **`docs/adr/`** 中与当前改动范围相关的 ADR，如果它存在。

如果这些文件当前还不存在，静默继续即可。不要把它们的缺失视为错误，也不要在开始工作前强制要求补齐。

## 文档布局

本仓库按 single-context 方式配置。

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 术语使用

当 `CONTEXT.md` 存在时，在 issue、设计说明、重构提案、问题假设或测试命名中，优先使用其中定义的领域术语。

## ADR 冲突处理

如果某个建议与已有 ADR 冲突，应显式指出冲突，而不是静默覆盖既有决策。
